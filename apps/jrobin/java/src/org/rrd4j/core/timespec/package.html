Time specification parsing and handling for JRobin RRD system. This package provides classes for parsing and handling time specifications used in Round Robin Database (RRD) systems. Time specifications define how time-based data should be interpreted, consolidated, and archived in RRD databases.

Key Components

  • {@link org.rrd4j.core.timespec.TimeSpec} - Time specification data structure
  • {@link org.rrd4j.core.timespec.TimeParser} - Parser for time specification strings
  • {@link org.rrd4j.core.timespec.Epoch} - Epoch time reference implementation

Main Features

  • Parse RRD time specification formats
  • Handle various time units (seconds, minutes, hours, days)
  • Support for absolute and relative time specifications
  • Integration with RRD consolidation algorithms
  • Time zone and epoch handling

Usage in RRD

Time specifications are used to define when RRD data points should be consolidated and archived. They allow for efficient storage of time series data by specifying the time intervals between data points.

Related Packages

  • {@link org.rrd4j.core} - Core JRobin RRD functionality
  • {@link org.rrd4j.data} - RRD data structures
