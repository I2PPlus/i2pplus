• Починаючи роботу
    Якщо ви тільки що почали I2P+, кількість активних пірів, зазначена в розділі Піри на бічній панелі, повинна збільшуватися протягом наступних декількох хвилин, а ви також побачите Локальне призначення з назвою Спільні клієнти в розділі Сервісні тунелі, а також можливо інших клієнтів і серверів в залежності від того, як сконфігуровано I2P. Ці Локальні призначення забезпечують підключення до сервісів на різних портах до мережі I2P, що дозволяє біторент, електронну пошту, веб-проксі та інші сервіси працювати. Ваша База даних мережі зазначає всі локально відомі піри в мережі. Ви також можете контролювати існуючі з'єднання з пірами, переглянути профілі продуктивності пірів та дивитися активні тунелі та їх стан.
  • Інтеграція в мережу
    Вперше запустивши I2P+ на інтеграцію (вбудовування) вас в мережу та пошуку додаткових пірів для оптимізації інтеграції може знадобитися кілька хвилин, тому будьте терплячими. Коли зірки зеленого кольору будуть позначені біля ваших Локальних призначень в розділі Сервісні тунелі на бічній панелі, існує багато чого, що можна зробити в I2P, а нижче ми розповімо вам про деякі з цих можливостей.
  • Стан мережі
    Коли I2P+ буде мати час для тестування підключення до мережі, показник стану мережі на бічній панелі ідеально повинен вказувати "OK". Якщо стану вказує "Firewalled", I2P+ все ще буде працювати, хоча продуктивність мережі не буде оптимальною, а ваша здатність надавати пропускну здатність мережі буде обмеженою. Для оптимальної продуктивності рекомендуємо відкрити зовнішній порт I2P+ у вашому брандмауері та маршрутизаторі Інтернет-провайдера. Більше інформації про повідомлення стану доступно на сторінці допомоги інтерфейсу користувача.
  • Підписки на адресну книгу
    Щоб I2P+ міг автоматично розв'язувати імена хостів .i2p, воно спирається переважно на підписки на сервіси, що забезпечують DNS-трансляцію для відомих хостів. Якщо ви встановили I2P+ замість оновлення з I2P, у вас вже будуть достатні підписки на адресну книгу; після встановлення ваша адресна книга повинна заповнитися іменами хостів для активних сервісів в мережі без додаткових дій. Якщо ви оновилися з I2P, перевірте розділ підписки на адресну книгу в ЧАПі для додавання додаткових підписок.
  • Налаштування браузера
    Щоб переглядати веб-сайти в мережі, вам потрібно налаштувати браузер для використання HTTP-проксі за замовчуванням, налаштованого на порт 4444, який також надає outproxy, що дозволяє анонімний перегляд сайтів звичайної мережі. На сторінці Налаштування в розділі Допомога наведено міні-посібник з налаштування Firefox, або перегляньте розділ нижче для посилання на посібник з налаштування для різних браузерів. Коли ви налаштуєте браузер і підписки на адресну книгу, notbob.i2p - це чудове місце для відкриття контенту в мережі.

Сервіси в I2P

  • Невидимий Інтернет та публічне переглядання веб-сторінок
    В I2P ви можете отримувати доступ до веб-сайтів, розміщених в I2P ("eepsites"), або анонімно переглядати звичайний Інтернет через вбудований в I2P веб-проксі (outproxy). Налаштуйте свій браузер для використання HTTP-проксі за адресою 127.0.0.1 порт 4444, потім перейдіть до прихованого веб-сайту в I2P або звичайної адреси http://. Щоб відключити або змінити доступ через outproxy до звичайного Інтернету, видаліть або змініть запис outproxy для false.i2p або purokishi.i2p в розділі Outproxies на сторінці редагування тунелю HTTP-проксі в Менеджері тунелів.
  • Анонімна електронна пошта
    Поштову систему Postman, побудовану на базі I2P, можна використовувати або через вбудований поштовий клієнт I2P (Susimail), або через звичайні поштові клієнти. Поштовий шлюз дозволяє отримувати пошту з звичайного Інтернету; вашу адресу @mail.i2p також можна контактувати через @i2pmail.org. Щоб отримати акаунт, відвідайте hq.postman.i2p.
  • Анонімна передача файлів
    I2PSnark інтегрований в I2P і забезпечує анонімні, зашифровані торрент-трансфери. Альтернативно, є BiglyBT (наступник Azureus/Vuze), клієнт біторрент, який підтримує торренти через I2P, і XD, клієнт біторренту, написаний на Go виключно для I2P. Додаткові сервіси включають браузерне хостинг файлів, такі як cake.i2p і fs.i2p.
  • Анонімний чат
    Запустіть ваш IRC-клієнт (наприклад, HexChat) і підключіться до сервера за адресою 127.0.0.1 порт 6668. Вам не потрібно налаштовувати проксі в IRC-клієнті; I2P надає вам локальний IRC-тунель (налаштований в вашому IRC-клієнті як IRC-сервер або віддалений хост). Якщо ви потрапили, канали #saltr і #i2p-dev - це лише два з доступних каналів, на які ви можете підписатися. Існує також крос-платформна за допомогою I2P месенджерна програма, доступна на Github.
  • Анонімне зашифроване веб-хостинг
    I2P включає веб-сервер для хостингу власного сайту в мережі I2P: екземпляр Jetty, що прослуховує http://127.0.0.1:7658/. Запустивши попередньо налаштований тунель I2P веб-сервера, ваш сайт буде видимий (але не знайдений) іншим. Інструкції по налаштуванню сайту та реєстрація .i2p домену наведені у розділі допомога.
  • Плагіни для I2P
    Відвідайте сторінку плагінів zzz для прямих посилань на завантаження кількох плагінів. Якщо ви розробник, надається фреймворк безвідносного до мови для написання власних плагінів.

Інформація для розробників та учасників

  • Повідомлення про помилки
    Щоб повідомити про помилку, будь ласка, створіть ticket на git.idk.i2p. Для розробницьких дискусій, будь ласка, відвідайте форуми розробників zzz або канал розробників (#i2p-dev) в IRC-мережі I2P.
  • Розробка на I2P
    Якщо ви розробник, варто розглянути можливість написання власних додатків для I2P або покращення безпеки додатку шляхом використання мережі I2P. Підтримуються як UDP, так і TCP протоколи, вони передбачені. Також можливий потік у реальному часі. Ми також надаємо повний доступ до Java API, а основні бібліотеки I2P розміщені в репозиторії Apache Maven. Розробники можуть переглядати початковий код на Github (I2P) або Gitlab (I2P+), і контролювати поточний стан збірок через Travis (I2P) чи Gitlab (I2P+). Для збирання з початкового коду потрібні Java SDK та Apache Ant.
      • Взяти участь!
        I2P розробляється і підтримується в основному завдяки волонтерській участі членів спільноти без фінансування. Більше інформації про те, як ви можете взяти участь, доступно на сторінці волонтерів на офіційному веб-сайті.
      • I2P+
        Для більш докладної інформації про I2P+, будь ласка, відвідайте skank.i2p або i2pplus.github.io, або приєднуйтесь до каналу #saltR в IRC-мережі I2P. Патчі для I2P+ запитані; будь ласка, надсилайте pull requests через Gitlab.
