• 起動
    I2P+を起動したばかりの場合、サイドバーの Peers セクションの下に表示される Active Peers の数が数分後に増えるはずです。また、サービス・トンネル セクションには 地元の目的地 として 共有クライアント という名前が表示され、I2Pの設定によっては他のクライアントやサーバも表示されることがあります。これらの 地元の目的地 は、I2Pネットワーク上のさまざまなポートに接続し、BitTorrentやメール、ウェブプロキシなどのサービスを利用できるようにします。あなたのネットワークデータベースには、ネットワーク上でローカルに知られているピアのすべてが表示されます。また、既存のピア接続をモニタリングしたり、ピアのパフォーマンスプロファイルを確認したり、アクティブなトンネルとその状態を表示することもできます。
  • ネットワーク統合
    初めてI2P+を起動すると、ネットワークに統合されるまでに数分かかる場合があります。統合の最適化のために追加のピアを見つけますので、しばらくお待ちください。サイドバーの サービス・トンネル の下にある 地元の目的地 に緑の星が表示されると、I2Pでできることがさまざまで、以下でいくつか紹介します。
  • ネットワークステータス
    I2P+がネットワーク接続をテストする時間があると、サイドバーのネットワークステータスインジケータは理想的には「OK」と表示されるはずです。ステータスが「Firewalled」を示している場合、I2P+は機能しますが、ネットワークのパフォーマンスは最適ではなく、ネットワークへの帯域幅の貢献能力も制限されます。最適なパフォーマンスを得るためには、ファイアウォールやISPが提供するルータで外部のI2P+ポートを開放することを検討してください。ステータスメッセージに関する詳細な情報は、ユーザーインターフェースのヘルプページで入手できます。
  • アドレス帳の購読
    I2P+が.i2pのホスト名を自動的に解決できるようにするためには、既知のホストのDNS変換を提供するサービスに対する購読を主に依存しています。I2Pからのアップグレードではなく、I2P+をインストールした場合、既に十分なアドレス帳の購読が設定されているはずです。インストール後、ネットワーク上のアクティブなサービスのホスト名が自動的にアドレス帳に表示されるはずです。I2Pからアップグレードした場合は、追加の購読を追加するためのヘルプは、アドレス帳の購読セクションにあります。
  • ブラウザの設定
    ネットワーク上のウェブサイトを閲覧するためには、ブラウザをデフォルトのHTTPプロキシに設定する必要があります。デフォルトのHTTPプロキシはポート4444で動作し、クリアネットサイトの匿名閲覧を許可するアウトプロキシも提供しています。ブラウザの設定についてのFirefoxのミニガイドは、ヘルプセクションの設定ページにあります。また、マルチブラウザの設定ガイドへのリンクも以下のセクションを参照してください。ブラウザとアドレス帳の購読を設定したら、notbob.i2pはネットワーク上のコンテンツを発見するための素晴らしいサイトです。

I2Pプラットフォームのサービス

  • 不可視インターネット＆パブリックウェブブラウジング
    I2Pでは、I2Pホスト型ウェブサイト（「イープサイト」）にアクセスしたり、I2Pの組み込みウェブプロキシ（アウトプロキシ）を介して通常のウェブを匿名で閲覧することができます。ブラウザを設定して、HTTPプロキシを127.0.0.1ポート4444で使用して、隠れたサービスウェブサイトまたは通常の非暗号化http://アドレスにアクセスしてください。通常のインターネットへのアウトプロキシアクセスを無効化または変更するには、HTTPプロキシクライアントトンネルの編集ページのアウトプロキシセクションでfalse.i2pまたはpurokishi.i2pのエントリを削除または変更してください。これはトンネル・マネージャーで行えます。
  • 匿名メール
    I2Pベースのメールシステムには、I2Pの組み込みメールクライアント（Susimail）または任意の標準メールクライアントからアクセスすることができます。メールゲートウェイを使用すると、通常のインターネットアドレスからメールを受信することができます。あなたの@mail.i2pアドレスは@i2pmail.orgでも連絡可能です。アカウントを取得するには、hq.postman.i2pを訪れてください。
  • 匿名ファイル転送
    I2Pには、匿名かつ暗号化されたBitTorrent転送を提供するI2PSnarkが統合されています。また、I2PでのトレントをサポートするBitTorrentクライアントであるBiglyBT（Azureus/Vuzeの後継）や、goで書かれたI2P専用のBitTorrentクライアントであるXDなども利用できます。追加のサービスには、cake.i2pやfs.i2pなどのブラウザベースのファイルホスティングもあります。
  • 匿名チャット
    IRCクライアントを起動して（例：HexChat）、サーバーに接続してください。接続先は127.0.0.1ポート6668です。IRCクライアントでプロキシを設定する必要はありません。I2PはローカルのIRCトンネルを提供します（IRCサーバーまたはリモートホストとしてIRCクライアントで設定済み）。接続後は、#saltrや#i2p-devなど、参加したい利用可能なチャンネルが何個かあります。また、クロスプラットフォームのI2Pベースのインスタントメッセンジャーアプリケーションもあり、Githubから入手できます。
  • 匿名暗号化ウェブホスティング
    I2Pにはウェブサーバーが付属しており、独自のウェブサイトをI2Pネットワーク上でホストすることができます。利用できるのはJettyのインスタンスで、ポートはhttp://127.0.0.1:7658/です。事前に設定されたI2P ウェブサーバー・トンネルを起動すると、他の人には見えますが（発見されません）、あなたのサイトを表示することができます。サイトの設定方法や.i2pドメインの登録手順については、ヘルプセクションに説明があります。
  • I2Pのプラグイン
    いくつかのプラグインの直接ダウンロードリンクについては、zzzのプラグインページをご覧ください。開発者の方には、独自のプラグインを作成するための言語に依存しないフレームワークも用意されています。

デベロッパーや貢献者のための情報

  • バグの報告
    バグを報告する場合は、git.idk.i2pでチケットを作成してください。開発者向けのディスカッションについては、zzzの開発者フォーラムまたはI2PのIRCネットワークの開発者チャンネル (#i2p-dev)をご覧ください。
  • I2Pでの開発
    開発者の方は、I2Pアプリケーションの作成や、I2Pネットワークを活用してアプリケーションのセキュリティを向上させることを検討してください。UDPプロトコルとTCPプロトコルの両方が提供されており、リアルタイムストリーミングも可能です。また、Java APIへの完全アクセスが提供され、I2PのコアライブラリはApache Mavenリポジトリでホストされています。開発者はGithub（I2P）またはGitlab（I2P+）でソースコードを閲覧し、現在のビルド状況はTravisの継続的統合プラットフォーム（I2P）またはGitlab（I2P+）で監視できます。ソースコードのビルドにはJava SDKとApache Antが必要です。
  • 参加しましょう！
    I2Pは、コミュニティメンバーによる非公的なボランティア活動によって主に開発および維持されています。参加方法の詳細については、公式ウェブサイトのボランティアページで入手できます。
  • I2P+
    I2P+に関する詳細は、skank.i2pまたはi2pplus.github.ioをご覧ください。または、I2P IRCネットワークのチャンネル#saltRに参加してください。I2P+のパッチは歓迎です。プルリクエストはGitlabを通じて送信してください。
