• Cominciando
    Se hai appena avviato I2P+, il numero di Peer Attivi indicato nella sezione Peer nella barra laterale dovrebbe aumentare nei prossimi minuti, e vedrai anche una Destinazione Locale chiamata Client Condivisi elencata nella sezione Tunnel di Servizio, e eventualmente altri client e server a seconda della configurazione di I2P. Queste Destinazioni Locali forniscono connessioni a servizi su diversi porti alla rete I2P, consentendo a BitTorrent, e-mail, proxy web e altri servizi di funzionare. Il tuo Database di Rete mostra tutti i peer localmente conosciuti nella rete. Puoi anche monitorare le connessioni dei peer esistenti, analizzare i profili di prestazioni dei peer e visualizzare i tunnel attivi e il loro stato.
  • Integrazione nella Rete
    La prima volta che avvii I2P+ potrebbe richiedere alcuni minuti per integrarti nella rete e trovare peer aggiuntivi per ottimizzare la tua integrazione, quindi sii paziente. Una volta che accanto alle tue Destinazioni Locali nella sezione Tunnel di Servizio della barra laterale saranno indicati asterischi verdi, ci sono diverse cose che puoi fare con I2P, e di seguito te ne presentiamo alcune.
  • Stato della Rete
    Una volta che I2P+ ha avuto il tempo di testare la tua connettività di rete, l'indicatore dello Stato della Rete nella barra laterale dovrebbe idealmente indicare "OK". Se lo stato indica "Firewalled", I2P+ funzionerà comunque, anche se le prestazioni di rete non saranno ottimali e la tua capacità di contribuire alla larghezza di banda della rete sarà ostacolata. Per prestazioni ottimali, considera l'apertura della tua porta esterna I2P+ sul firewall e sul router fornito dal tuo ISP. Ulteriori informazioni sui messaggi di stato sono disponibili nella pagina di aiuto dell'interfaccia utente.
  • Sottoscrizioni della Rubrica Indirizzi
    Affinché I2P+ possa risolvere automaticamente gli hostname .i2p, si basa principalmente su sottoscrizioni a servizi che forniscono la traduzione DNS per gli host conosciuti. Se hai installato I2P+ anziché eseguire l'aggiornamento da I2P, avrai già sufficienti sottoscrizioni della rubrica indirizzi configurate; dopo l'installazione, la tua rubrica indirizzi dovrebbe popolarsi con gli hostname dei servizi attivi nella rete senza interventi. Se hai eseguito l'aggiornamento da I2P, controlla la sezione delle sottoscrizioni della rubrica indirizzi nel FAQ per aiuto nell'aggiunta di ulteriori sottoscrizioni.
  • Configurazione del browser
    Per navigare sui siti web nella rete, sarà necessario configurare il tuo browser per utilizzare il proxy HTTP predefinito configurato sulla porta 4444, che fornisce anche un outproxy che consente la navigazione anonima dei siti Clearnet. Una mini-guida alla configurazione di Firefox è fornita nella pagina di configurazione nella sezione di aiuto, oppure fare riferimento alla sezione sottostante per un link alla guida di configurazione multi-browser. Una volta configurato il browser e le sottoscrizioni degli elenchi degli indirizzi, notbob.i2p è un ottimo sito per scoprire contenuti nella rete.

Servizi su I2P

  • Invisible Internet e navigazione web pubblica
    Su I2P è possibile accedere a siti web ospitati su I2P ("eepsites") o navigare nel web normale in modo anonimo tramite il proxy web integrato in I2P (outproxy). Configura il tuo browser per utilizzare il proxy HTTP a 127.0.0.1 porta 4444, quindi naviga verso un sito web di servizio nascosto o un indirizzo normale non crittografato http://. Per disabilitare o cambiare l'accesso all'internet normale tramite outproxy, rimuovi o modifica l'ingresso dell'outproxy per false.i2p o purokishi.i2p nella sezione Outproxies nella pagina di modifica del tunnel client proxy HTTP nel Gestore del tunnel.
  • E-mail anonima
    Il sistema di posta basato su I2P di Postman può essere accessibile tramite il client di posta integrato di I2P (Susimail) o qualsiasi client di posta standard. Un gateway di posta consente di ricevere messaggi da indirizzi internet normali; il tuo indirizzo @mail.i2p può essere contattato anche tramite @i2pmail.org. Per un account, visita hq.postman.i2p.
  • Trasferimento anonimo di file
    I2PSnark è integrato in I2P, fornendo trasferimenti BitTorrent anonimi e crittografati. In alternativa, c'è BiglyBT (il successore di Azureus/Vuze), un client BitTorrent che supporta il torrenting su I2P, e XD, un client BitTorrent esclusivo per I2P scritto in go. I servizi aggiuntivi includono l'hosting di file basato sul browser come cake.i2p e fs.i2p.
  • Chat anonima
    Avvia il tuo client IRC (ad esempio HexChat) e connettiti al server su 127.0.0.1 porta 6668. Non è necessario configurare un proxy nel tuo client IRC; I2P ti fornisce un tunnel IRC locale (configurato nel tuo client IRC come un server IRC o un host remoto). Una volta lì, #saltr e #i2p-dev sono solo un paio dei canali disponibili ai quali potresti voler partecipare. C'è anche un'applicazione di messaggistica istantanea basata su I2P, disponibile su Github.
  • Hosting web crittografato anonimo
    I2P include un server web per ospitare il tuo sito web sulla rete I2P: un'istanza di Jetty in ascolto su http://127.0.0.1:7658/. Dopo aver avviato il Tunnel del server web I2P preconfigurato, il tuo sito sarà visibile (ma non individuabile) agli altri. Le istruzioni per la configurazione del tuo sito e la registrazione di un dominio .i2p sono fornite nella sezione di aiuto.
  • Plugin per I2P
    Visita la pagina dei plugin di zzz per scaricare direttamente più plugin. Se sei uno sviluppatore, è fornito un framework indipendente dal linguaggio per scrivere i tuoi plugin.

Informazioni per Sviluppatori e Collaboratori

  • Segnalazione Bug
    Per segnalare un bug, per favore apri un ticket su git.idk.i2p. Per discussioni relative agli sviluppatori, per favore visita i forum degli sviluppatori di zzz o il canale degli sviluppatori (#i2p-dev) nella rete IRC di I2P.
  • Sviluppo su I2P
    Se sei uno sviluppatore, potresti considerare di scrivere le tue applicazioni I2P o migliorare la sicurezza della tua applicazione sfruttando la rete I2P. Sia i protocolli UDP che TCP sono provisionati e lo streaming in tempo reale è anche possibile. Forniamo inoltre pieno accesso all'API Java, e le librerie I2P core sono ospitate sul repositorio Apache Maven. Gli sviluppatori possono navigare il codice sorgente su Github (I2P) o su Gitlab (I2P+), e monitorare lo stato di compilazione attuale tramite la piattaforma di integrazione continua Travis (I2P) o tramite Gitlab (I2P+). Per compilare il codice sorgente sono richiesti il Java SDK e Apache Ant.
  • Partecipa!
    I2P è sviluppato e mantenuto principalmente attraverso la partecipazione volontaria e non retribuita dei membri della comunità. Ulteriori informazioni su come puoi partecipare sono disponibili nella pagina dei volontari sul sito ufficiale.
  • I2P+
    Per ulteriori informazioni su I2P+, visita skank.i2p o i2pplus.github.io, oppure unisciti al canale #saltR sulla rete IRC di I2P. I suggerimenti per I2P+ sono benvenuti; invia i tuoi pull request tramite Gitlab.
