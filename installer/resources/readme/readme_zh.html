• 启动成功！
    如果您刚刚启动 I2P，在左侧面板的对等节点部分下显示的活动对等节点数量将在接下来的几分钟开始增长。 可以看到那里列出的一个被称为共享隧道客户端的本地隧道，或许还有其他客户端和服务器。 如果没有，参见下方的故障排除部分。 这些本地隧道提供到 I2P 网络的连接，让你您可以使用BT下载，电子邮件，网页浏览和其他服务。 您的网络数据库展示了网络上全部已知的对等节点。 此外，您可以查看节点信息，并查看隧道来了解路由器的运行状态。 帮助页面有更多信息。
  • 网络整合
    当您启动 I2P后，可能需要几分钟的时间来引导（整合）你的路由到网络并寻找其他的对等节点，所以请耐心等待。（中国用户请考虑使用代理补种） 在正常工作期间，侧面板的 I2P 隧道构建指示器可能显示“拒绝共享隧道”；这是正常的，中国是严格国家，一般情况下我们不会让你为网络贡献流量。 一旦在您的本地隧道旁显示绿星，您就可以使用 I2P了 ，下面我们为您介绍一些可以做的事：

I2P 服务

  • I2P网站和表网浏览
    在 I2P 上您可以访问I2P网站（“Eepsite”）及其他服务，并且您可以托管您自己的网站。 您也可以通过 I2P 内置的表网代理（出口代理）匿名访问普通网络。 配置您的浏览器使用 127.0.0.1:4444的 HTTP 代理，然后浏览I2P内部网站或普通网站。 如果您希望禁用或更改访问普通网络的出口代理，移除或修改隧道编辑器中 I2P HTTP 代理的出口代理选项。 在下方“有趣的网站”部分，我们列出了一些托管在 I2P 上的网站。
  • 匿名电子邮件
    Postman 托管的基于 I2P 的电子邮件可以通过 I2P的内置邮件客户端或任何外部邮件客户端访问。 访问 hq.postman.i2p 获取账户，账户可以向普通网络发送和接收邮件。 另外， I2P-Bote 是 I2P 内部的无服务器，安全（端到端加密）电子邮件系统，具有集成的匿名/假名回邮器，提供网页界面和内置地址簿。（已停止开发） 它可以作为插件安装。
  • 匿名文件传输
    I2PSnark 已集成在 I2P 内，提供匿名加密的 BitTorrent 传输。
  • 匿名聊天
    启动您的 IRC 客户端（如 HexChat）并连接到 127.0.0.1 6668 端口上的 IRC2P 网络。 您无需为您的 IRC 客户端配置代理；I2P 向您提供了本地 IRC 隧道（在您的 IRC 客户端中显示为 IRC服务器或远程主机）。 在 localhost:6668 上的您的本地 I2P IRC 隧道将指向由 Postman，echelon 和 zl0 托管在 I2P 上的三个 IRC 服务器中的一个，但您和他们都不知道对方在哪，并且您的通信在中转时将由 I2P 加密以阻止窃听。 当您完成后，来#i2p-chat 和 #i2p-dev 看看！ 也有一个基于 I2P 的I2PChat用于不受监控审查的匿名聊天，以及其他人托管的 IRC 服务器，Jabber 服务器，基于网站的聊天等等可以寻找。 您可以以任何您选择的方式通过 I2P 运行自己的服务，如果您是开发者，也编写您自己的 I2P 应用。 I2P同时支持UDP和TCP。 我们也提供了 Java API 的完全访问。
  • 论坛和博客
    所有普通博客，论坛，CMS 软件能在 I2P 上运行，但我们建议你设置时要特别注意安全，并保持所有相关的软件（如 PHP，MySQL，Python 和 Apache）最新！ 此外，I2P 上还有不少各种语言的社区运行的论坛；参见下方获取一些建议。
  • I2P 插件
    通过安装插件扩展 I2P 的可用性：博客，聊天，文件分享和其他插件已被编写和移植，等待您的安装！ 文档提供了编写您自己的插件的完整框架。

在 I2P 上托管服务

  • 内置网络服务器
    I2P 内置网络服务器供您在 I2P 网络上托管您自己的I2P网站（“匿名网站”）： 一个 Jetty 实例监听 http://127.0.0.1:7658/。 为托管您自己的内容，只需将您的文件放在 /eepsite/docroot/ 文件夹中（或将任何标准 JSP/Servlet .war 文件放在 eepsite/webapps 下，或将标准 CGI scripts 放在 eepsite/cgi-bin 下即可）。 您也可以运行其他网络服务器来替换内置的服务器，或者在另一个服务端隧道运行。

故障排除及更多帮助

  • 保持耐心！
    I2P 在您第一次运行时可能需要花费很长时间整合进网络因为它在引导并寻找对等节点。 您的 I2P 路由运行时间越长，性能越好，所以尝试并保持您的路由运行时间尽可能长，如果可能 24/7 最好！ 如果 30 分钟后，您的活动节点数量仍然少于 10 个对等节点，或已整合的对等节点总数少于 5，您可以做几件事情以检查问题：
  • 检查您的配置和带宽分配
    当您的带宽配置部分配置准确时，I2P 性能最好。 I2P 默认配置带宽相当保守，并不适合许多用例，所以请花时间查看这些设置并在必要时修正。 您分配的带宽，特别是上行带宽越多，您从网络获益越多。
  • 防火墙，调制解调器和路由器
    如果可能，请相应地配置您的调制解调器/路由器/电脑的防火墙以确保允许 I2P/Java 从网络不受到防火墙限制的访问。 如果您在过高的防火墙后但有不受限制的出站访问权限，I2P 仍能工作；您可以关闭入站访问并依赖 SSU IP 地址探测进行（防火墙穿透）将您连接到网络，您的侧面板上的网络状态将显示“网络：受到防火墙限制”。 为优化性能，请确保 I2P 的外部端口可从互联网访问。 参见下方详细信息。
  • 检查您的代理设置
    如果您无法访问任何网站（甚至无法访问 i2p-projekt.i2p），确保您的浏览器代理设置为 127.0.0.1:4444,类型HTTP代理。 如果您需要帮助，这里有一份配置您的浏览器使用 I2P 的指南。
  • 检查您的日志
    日志或许可以帮助解决问题。 您可能希望在我们的论坛上粘贴部分日志以获取帮助，或者可以粘贴引用链接并在 IRC 上以获取帮助。
  • 验证 Java 是最新版
    确保您的 Java 是最新版。 要求 7 或更高版本；推荐 8 或更高版本。 在日志页面顶部检查您的 Java 版本。
  • 在旧式硬件上运行的问题
    [Linux/BSD] 如果您无法通过 i2p/i2prouter start 启动路由，尝试同一文件夹下的 runplain.sh 脚本。 完全不需要使用 root 权限运行 I2P。 如果您需要编译 jBigi 库（极少数情况下需要），请查阅文档，访问论坛，或访问我们的 IRC 开发者频道。
  • 启用通用即插即用 (UPnP)
    您的调制解调器或路由器可能支持 UPnP 自动端口转发。 确保在您的调制解调器或路由器启用了 UPnP 支持。 然后重启 I2P 路由。 一旦 I2P 路由完成初始连接测试，如果成功，I2P 应报告“网络：正常”。（中国用户也应该打开UPnP，虽然这并不能保证你不会被防火长城限制入站连接）
  • 端口转发
    在您的调制解调器，路由器和/或防火墙上打开 I2P 的端口以获得更好的性能（中国用户请关闭TCP，这可能导致你的流量被探测）。 关于如何端口转发的更多信息可以很容易通过网络搜索找到。 如果您有任何困难，你可以在下面列出的我们的论坛和 IRC 频道寻求帮助。 注意 I2P 不支持通过 http 或 socks 代理连接到互联网，不过一旦连接到网络，您可以通过 I2P 本身连接到出口代理，但它们是志愿者运行的服务，不是I2P计划的一部分。
  • 获取在线支持
    您可能需要在 I2P 网站上获取信息，在 I2P 讨论论坛上发布消息或在 IRC2P 网络上的 #i2p 或 #saltR 上咨询。 这些频道也可以通过 Freenode IRC 在 I2P 网络外访问。
  • 报告错误
    请在 git.idk.i2p，或通过明网的i2pgit.org 上提交 ticket 以报告错误。 对于开发相关的讨论，请访问 zzz 的开发者论坛 或 IRC2P 网络上的开发者频道。 开发者可在 Github 上浏览源代码。
  • 参与进来！
    I2P的开发和维护主要通过社区成员的无偿自愿参与。关于如何参与的更多信息，可在官方网站的志愿者页面上找到。
  • I2P+
    关于I2P+的更多信息请查看 skank.i2p 或 i2pplus.github.io, 或者加入 IRC2P 上的 #saltR 频道。 欢迎给I2P+共享代码，请通过以下方式提交合并请求： Gitlab 或 git.i2p。
