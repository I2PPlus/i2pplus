• Запуск
    Если вы только что запустили I2P+, количество Активных пиров, указанное в разделе Пиры на боковой панели, должно начать увеличиваться в течение ближайших нескольких минут. Вы также увидите Локальный пункт назначения с названием Общие клиенты в разделе Сервисные туннели и, возможно, другие клиенты и серверы в зависимости от настроек I2P. Эти Локальные пункты назначения обеспечивают соединение с сервисами на разных портах I2P-сети, обеспечивая работу BitTorrent, электронной почты, веб-прокси и других сервисов. Ваша Сетевая база данных показывает всех известных локально пиров в сети. Вы также можете отслеживать существующие соединения с пирами, просматривать профили работы пиров и просматривать активные туннели и их статус.
  • Интеграция в сеть
    При первом запуске I2P+ может потребоваться несколько минут для запуска (интеграции) вас в сеть и поиска дополнительных пиров для оптимизации вашей интеграции, поэтому пожалуйста, будьте терпеливы. Когда рядом с вашими Локальными пунктами назначения в разделе Сервисные туннели на боковой панели появятся зеленые звездочки, у вас будет множество возможностей для работы с I2P, и далее мы познакомим вас с некоторыми из них.
  • Статус сети
    Когда I2P+ протестирует ваше сетевое подключение, индикатор Статуса сети на боковой панели должен показать "OK". Если статус указывает "Firewalled", I2P+ все равно будет работать, однако производительность сети будет неоптимальной, и ваша способность предоставлять пропускную способность для сети будет ограничена. Для обеспечения оптимальной производительности, рекомендуется открыть внешний I2P+ порт в вашем брандмауэре и маршрутизаторе, предоставленном вашим интернет-провайдером. Дополнительная информация о сообщениях о статусе доступна на странице справки пользовательского интерфейса.
  • Подписки в адресной книге
    Для того чтобы I2P+ мог автоматически разрешать .i2p имена хостов, он в основном полагается на подписки на сервисы, предоставляющие перевод DNS для известных хостов. Если вы установили I2P+ вместо обновления с I2P, у вас уже должно быть достаточно установленных подписок в адресной книге; после установки, ваша адресная книга должна заполняться именами хостов для активных сервисов в сети без вашего участия. Если вы обновились с I2P, ознакомьтесь с разделом подписок в адресной книге в часто задаваемых вопросах для получения помощи по добавлению дополнительных подписок.
  • Настройка браузера
    Чтобы просматривать веб-сайты в сети, вам необходимо настроить ваш браузер для использования стандартного HTTP-прокси, настроенного на порту 4444, который также предоставляет аутпрокси, позволяющий анонимно просматривать сайты в сети. На странице Настройки в разделе справки предоставляется мини-руководство по настройке Firefox или обратитесь к разделу ниже для ссылки на руководство по настройке мульти-браузера. После настройки браузера и подписок в адресной книге, notbob.i2p - отличный сайт для поиска контента в сети.

Услуги на платформе I2P

  • Невидимый интернет и публичный веб-браузинг
    На I2P вы можете получить доступ к веб-сайтам I2P ("eepsites") или анонимно просматривать обычный веб через встроенный веб-прокси I2P (аутпрокси). Настройте ваш браузер для использования HTTP-прокси на 127.0.0.1, порт 4444, а затем перейдите на веб-сайт скрытой услуги или обычный незашифрованный адрес http://. Чтобы отключить или изменить доступ к обычному Интернету через аутпрокси, удалите или измените запись аутпрокси для false.i2p или purokishi.i2p в разделе Outproxies на странице редактирования клиента HTTP Proxy tunnel в Менеджере туннелей I2P.
  • Анонимная электронная почта
    Система электронной почты на базе I2P поддерживает доступ через встроенный почтовый клиент I2P (Susimail) или любой стандартный почтовый клиент. Шлюз электронной почты позволяет получать письма с обычных адресов Интернета; ваш адрес @mail.i2p также можно использовать по адресу @i2pmail.org. Для создания учетной записи посетите hq.postman.i2p.
  • Анонимная передача файлов
    В I2P интегрирована I2PSnark, предоставляющая анонимную, зашифрованную передачу файлов по BitTorrent. В качестве альтернативы также можно использовать BiglyBT (преемник Azureus/Vuze) — клиент BitTorrent, поддерживающий загрузку торрентов через I2P, и XD — клиент BitTorrent, работающий только в I2P и написанный на Go. Дополнительные услуги включают браузерное размещение файлов, такие как cake.i2p и fs.i2p.
  • Анонимный чат
    Запустите свой клиент IRC (например, HexChat) и подключитесь к серверу на 127.0.0.1, порт 6668. Вам не нужно настраивать прокси в своем клиенте IRC; I2P предоставляет вам локальный туннель IRC (настраивается в вашем клиенте IRC как IRC-сервер или удаленный хост). Как только вы попадете туда, #saltr и #i2p-dev будут только несколько доступных каналов, в которых вы можете присоединиться. Есть также кросс-платформенное приложение мгновенного обмена сообщениями на базе I2P, доступное на Github.
  • Анонимный зашифрованный хостинг веб-сайтов
    I2P поставляется с веб-сервером для размещения собственного веб-сайта в сети I2P: экземпляр Jetty, слушающий на http://127.0.0.1:7658/. После запуска предварительно настроенного туннеля сервера I2P ваш сайт станет видимым (но не обнаруживаемым) для других. Инструкции по настройке сайта и регистрации домена .i2p предоставляются в разделе Справка.
  • Плагины для I2P
    Посетите страницу плагинов zzz для прямых ссылок на загрузку нескольких плагинов. Если вы разработчик, предоставляется многофункциональная среда для написания собственных плагинов.

Информация для разработчиков и участников

  • Сообщение об ошибках
    Для сообщения об ошибке, пожалуйста, создайте тикет на github.com/I2PPlus/i2pplus/issues. Для обсуждения разработчиками, пожалуйста, посетите форум разработчиков zzz или канал разработчиков (#i2p-dev) в IRC-сети I2P.
  • Разработка на I2P
    Если вы являетесь разработчиком, вы можете написать своё собственное приложение I2P или улучшить безопасность вашего приложения, используя сеть I2P. Используются как UDP, так и TCP протоколы, и реальное стриминговое вещание также возможно. Мы также предлагаем полный доступ к Java API, а основные библиотеки I2P размещены в репозитории Apache Maven. Разработчики могут просматривать исходный код на Github (I2P) или Gitlab (I2P+), а также отслеживать текущий статус сборки через платформу непрерывной интеграции Travis (I2P) или через Gitlab (I2P+). Для сборки из исходного кода требуется Java SDK и Apache Ant.
  • Участвуйте!
    I2P разрабатывается и поддерживается преимущественно благодаря добровольному участию членов сообщества. Дополнительная информация о возможности участия доступна на странице волонтеров на официальном веб-сайте.
  • I2P+
    Дополнительная информация о I2P+ доступна на skank.i2p или i2pplus.github.io, или присоединитесь к каналу #saltR в IRC-сети I2P. Патчи для I2P+ приветствуются; пожалуйста, отправляйте запросы на слияние через Gitlab.
