• 시작하기
    I2P+를 처음 시작한 경우 활성 피어 아래 측바의 피어 섹션에 표시된 숫자가 다음 몇분 내로 성장할 것이고 서비스 터널 섹션 아래 공유 클라이언트라는 이름의 로컬 목적지가 나열될 것입니다. I2P가 어떻게 구성되어 있는지에 따라 다른 클라이언트와 서버도 나열될 수 있습니다. 이 로컬 목적지는 I2P 네트워크에 연결된 서로 다른 포트를 통해 서비스를 제공하며, 비트토렌트, 이메일, 웹 프록시 등의 서비스를 활성화합니다. 네트워크 데이터베이스는 네트워크상에서 알려진 모든 로컬 피어를 표시합니다. 또한 진행 중인 피어 연결을 모니터하고, 피어 성능 프로필을 검토하며, 활성 터널과 그 상태를 확인할 수도 있습니다.
  • 네트워크 연계
    I2P+를 처음 시작할 때는 네트워크에 통합되고 추가 피어를 찾아 네트워크 통합을 최적화하는 데에 몇분이 걸릴 수 있으니 인내심을 가져 주세요. 측바의 서비스 터널 섹션 아래 로컬 목적지 옆에 녹색 별표가 표시된다면 I2P에서 다양한 작업을 수행할 수 있습니다. 아래에서 I2P의 일부 기능을 소개합니다.
  • 네트워크 상태
    I2P+가 네트워크 연결 가용성을 테스트한 후에 측바의 네트워크 상태 표시기는 "정상"으로 표시될 것입니다. 상태가 "방화벽"으로 표시된다면 I2P+는 계속 작동하지만 성능이 최상이 아닐 것이고 네트워크 대역폭 공헌 역량이 제한될 것입니다. 최적의 성능을 원한다면 I2P+ 외부 포트를 방화벽과 ISP 제공 라우터에서 열어주세요. 상태 메시지의 의미는 사용자 인터페이스 도움말 페이지를 참조하세요.
  • 주소책 구독
    I2P+가 .i2p 도메인을 자동으로 확인할 수 있도록하려면 주로 DNS-대-호스트 서비스에 구독에 의존합니다. I2P를 설치한 경우 기본적으로 충분한 주소책 구독이 구성됩니다. 설치 후 주소책에는 활성 네트워크 서비스의 호스트명이 사전에 나타날 것입니다. I2P에서 업그레이드한 경우 FAQ의 주소책 구독 섹션을 확인하고 추가 구독을 설정하세요.
  • 브라우저 설정
    네트워크에서 웹사이트를 검색하려면 기본 HTTP 프록시를 포트 4444로 설정하도록 브라우저를 설정해야합니다. 이는 동시에 익명 웹 검색을 허용하는 아웃프록시를 제공합니다. Firefox 설정 지침(미니 가이드)이 도움 섹션의 구성 페이지에 제공되며, 아래에는 다중 브라우저 설정 가이드 링크가 있습니다. 브라우저와 주소책 구독을 설정한 후 notbob.i2p는 네트워크 콘텐츠를 발견하는 좋은 사이트입니다.

I2P 서비스

  • 불투명 인터넷 및 공개 웹 검색
    I2P에서 I2P에 호스팅되는 웹사이트("이이프사이트")를 볼 수 있습니다. 또는 I2P 내장 웹 프록시(아웃프록시)를 통해 익명으로 법외 인터넷을 검색할 수 있습니다. 브라우저 설정하여 127.0.0.1 포트 4444의 HTTP 프록시 사용하세요. 그리고 은밀 서비스 사이트나 일반적인 http:// 주소로 이동하세요. 법외 인터넷 액세스를 비활성화하거나 변경하려면 터널 관리자의 HTTP 프록시 클라이언트 터널 편집 페이지의 아웃프록시 섹션의 false.i2p 또는 purokishi.i2p 항목을 제거하거나 수정하세요.
  • 익명 이메일
    포스티움의 I2P 기반 메일 시스템은 I2P 내장 메일 클라이언트(수시메일)나 표준 메일 클라이언트로 액세스할 수 있습니다. 메일 게이트웨이를 통해 법외 주소에서 받은 메일도 받을 수 있습니다. @mail.i2p 주소는 @i2pmail.org로도 연락할 수 있습니다. 계정을 만들려면 hq.postman.i2p 방문하세요.
  • 익명 파일 전송
    I2PSnark는 I2P에 통합되어 익명 암호화 비트토렌트 전송을 제공합니다. 대안으로 BiglyBT(아자우루스/부즈의 후속 버전)가 I2P 위에서 비트토렌트 지원하며, XD는 고로 만든 I2P 전용 비트토렌트 클라이언트입니다. 브라우저 기반 파일 호스팅인 cake.i2p, fs.i2p도 있습니다.
  • 익명 채팅
    IRC 클라이언트(HexChat 등) 실행하고 서버 127.0.0.1 포트 6668에 연결하세요. IRC 클라이언트에서 프록시 설정 필요 없습니다. I2P에서 귀하에게 로컬 IRC 터널(IRC 클라이언트에서 서버 또는 원격 호스트로 설정)을 제공합니다. 들어가면 #saltr, #i2p-dev 등 가입할 수 있는 채널이 많습니다. 크로스플랫폼 I2P 기반 즉시 메시지 애플리케이션도 깃허브에서 제공합니다.
  • 익명 암호화 웹 호스팅
    I2P에는 자신의 웹사이트를 I2P 네트워크에서 호스팅할 수 있는 웹 서버가 내장되어 있습니다. 제티 인스턴스가 http://127.0.0.1:7658/에 감시하고 있습니다. 미리 구성된 I2P 웹서버 터널을 시작하면 사이트가 다른 사람에게 보입니다(검색되지 않습니다). 웹사이트 설정 방법과 .i2p 도메인 등록 절차는 도움 섹션에서 제공됩니다.
  • I2P 플러그인
    수많은 플러그인 직접 다운로드 링크가 제공되는 zzz의 플러그인 페이지 방문하세요. 개발자이시면 언어 독립적 프레임워크가 자신의 플러그인 개발을 제공합니다.

개발자 및 기여자를 위한 정보

  • 버그 보고
    버그를 보고하려면 git.idk.i2p에서 티켓을 제출하세요. 개발자 관련 토의는 zzz의 개발자 포럼이나 I2P IRC 네트워크의 개발자 채널(#i2p-dev) 방문하세요.
  • I2P 상 개발
    개발자인 경우 I2P 애플리케이션 개발이나 I2P 네트워크를 활용하여 애플리케이션 보안 강화를 고려하세요. UDP와 TCP 프로토콜이 지원되며, 실시간 스트리밍도 가능합니다. 자바 API 접근도 제공하며, 핵심 라이브러리는 아파치 메이븐 저장소에 호스팅됩니다. 소스는 깃허브(I2P)와 깃랩(I2P+)에서, 트래비스(I2P)와 깃랩(I2P+)을 통해 빌드 상태 확인 가능합니다. 소스 빌드를 위해 자바 SDK와 아파치 앤트 필요합니다.
  • 참여하기!
    I2P는 주로 무보증 자원 기여를 통해 개발과 유지 관리됩니다. 참여 방법은 공식 웹사이트 자원 섹션에서 확인하세요.
  • I2P+
    더 자세한 I2P+ 정보는 skank.i2p나 i2pplus.github.io를 방문하거나 I2P IRC 네트워크 #saltR 채널 참여하세요. I2P+ 패치 환영합니다. 깃랩 풀 요청으로 제출하세요.
