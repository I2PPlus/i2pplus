• Att starta upp
    Om du precis har startat I2P+, kommer antalet Aktiva Nodpar som visas under avsnittet Nodpar på sidopanelen börja öka under de närmaste minuterna, och du kommer även se en Lokal Destination vid namn Delade Klienter listad under avsnittet Tjänstetunnlar, och eventuellt andra klienter och servrar beroende på hur I2P är konfigurerat. Dessa Lokala Destinationer ger anslutningar till tjänster på olika portar till I2P-nätverket, vilket gör att BitTorrent, e-post, webbproxy och andra tjänster kan fungera. Din Nätverksdatabas visar alla lokalt kända noder i nätverket. Du kan också övervaka befintliga Nodanslutningar, granska Nodpekningars prestandaprofiler och visa aktiva tunnlar och deras status.
  • Nätverksintegration
    Första gången du startar I2P+ kan det ta några minuter att starta upp (integrera) dig i nätverket och hitta ytterligare noder för att optimera din integrering, så var vänlig och ha tålamod. När gröna stjärnor visas bredvid dina Lokala Destinationer under avsnittet Tjänstetunnlar på sidopanelen, finns det olika saker du kan göra med I2P, och nedan introducerar vi dig till några av dem.
  • Nätverksstatus
    När I2P+ har haft tid att testa din nätverksanslutning, bör indikatorn för Nätverksstatus på sidopanelen ideally visa "OK". Om statusen visar "Firewalled", kommer I2P+ fortfarande att fungera även om nätverksprestandan inte blir optimal, och din förmåga att bidra med bandbredd till nätverket kommer att vara hindrad. För optimal prestanda, överväg att öppna din externa I2P+-port i din brandvägg och router från din internetleverantör. Mer information om statusmeddelandena finns på hjälpsidan för Användargränssnitt.
  • Adressboksprenumerationer
    För att I2P+ ska kunna lösa .i2p-hostnamn automatiskt, förlitar det sig i första hand på prenumerationer till tjänster som tillhandahåller DNS-översättning för kända värdar. Om du installerade I2P+ istället för att uppgradera från I2P, kommer du redan ha tillräckliga adressboksprenumerationer konfigurerade; efter installationen kommer din adressbok att fyllas med hostnamn för aktiva tjänster på nätverket utan att du behöver göra något. Om du har uppgraderat från I2P, kontrollera avsnittet för prenumerationer i adressboken i FAQ för hjälp med att lägga till ytterligare prenumerationer.
  • Webbläsarkonfiguration
    För att kunna surfa på webbplatserna i nätverket måste du konfigurera din webbläsare att använda standard-HTTP-proxy som är konfigurerad på port 4444, vilket också ger en utproxy som tillåter anonym surfning på klarwebbplatser. En miniguide för att konfigurera Firefox finns på Konfigurationssidan i hjälpavsnittet, eller se nedan för en länk till guiden för inställningar för flera webbläsare. När du har konfigurerat din webbläsare och adressboksprenumerationer, är notbob.i2p en bra webbplats för att upptäcka innehåll på nätverket.

Tjänster på I2P

  • Osynlig Internet och offentlig webbsökning
    På I2P kan du få tillgång till I2P-härdade webbplatser ("eepsites") eller bläddra på den normala webben anonymt via I2P's inbyggda webbproxy (outproxy). Konfigurera din webbläsare för att använda HTTP-proxyn på 127.0.0.1 port 4444, sedan bläddra till en dold tjänstwebbplats eller en normal, okrypterad http://-adress. För att inaktivera eller ändra utproxtillgång till den normala internet, ta bort eller ändra outproxy-inmatningen för false.i2p eller purokishi.i2p under avsnittet Outproxies på redigera sidan för HTTP Proxy-klienttunneln i Tunnel Manager.
  • Anonym e-post
    Postmans I2P-baserade mejlsystem kan nås antingen via I2P's inbyggda mejlklient (Susimail) eller någon standard mejlklient. En mejlport ger möjlighet att ta emot mejl från vanliga internetadresser; din @mail.i2p-adress kan också kontaktas via @i2pmail.org. För ett konto besök hq.postman.i2p.
  • Anonym filöverföring
    I2PSnark är integrerad i I2P och möjliggör anonyma, krypterade BitTorrent-överföringar. Alternativt finns det BiglyBT (efterträdaren till Azureus / Vuze), en BitTorrent-klient som stöder torrentering över I2P, och XD, en I2P-endast BitTorrent-klient skriven i go. Ytterligare tjänster inkluderar webbläsarbaserad filhosting som cake.i2p och fs.i2p.
  • Anonym chatt
    Starta din IRC-klient (t.ex. HexChat) och anslut till servern på 127.0.0.1 port 6668. Du behöver inte konfigurera en proxy i din IRC-klient; I2P tillhandahåller dig en lokal IRC-tunnel (konfigurerad i din IRC-klient som en IRC-server eller fjärrvärd). När du är där, #saltr, och #i2p-dev är bara ett par av de tillgängliga kanalerna du kan gå med i. Det finns också en plattformsoberoende I2P-baserad direktmeddelandeapplikation, tillgänglig på Github.
  • Anonym krypterad webbhosting
    I2P levereras med en webbserver för att hosta din egen webbplats på I2P-nätverket: en Jetty-instans som lyssnar på http://127.0.0.1:7658/. Efter att ha startat den förkonfigurerade I2P Webserver Tunnel, kommer din webbplats vara synlig (men inte upptäckbar) för andra. Instruktioner för att konfigurera din webbplats och registrera ett .i2p-domän tillhandahålls i hjälpavsnittet.
  • Plugins för I2P
    Besök zzz's pluginsida för direkt nedladdningslänkar till flera plugins. Om du är utvecklare finns ett språkagnostiskt ramverk för att skriva dina egna plugins.

Information för utvecklare och bidragsgivare

  • Rapportera buggar
    För att rapportera en bugg, vänligen skapa en biljett på github.com/I2PPlus/i2pplus/issues. För diskussioner relaterade till utveckling, besök zzz's utvecklarforum eller utvecklar­kanalen (#i2p-dev) på I2P:s IRC-nätverk.
  • Utveckla på I2P
    Om du är en utvecklare kan du överväga att skriva dina egna I2P-applikationer eller förbättra säkerheten för din applikation genom att använda I2P-nätverket. Både UDP- och TCP-protokollen är tillgängliga, och realtids-strömning är också möjlig. Vi tillhandahåller också full tillgång till Java API, och kärnbiblioteken för I2P är värdade på Apache Maven-repot. Utvecklare kan bläddra genom källkoden på Github (I2P) eller Gitlab (I2P+), och övervaka den aktuella byggstatusen via Travis kontinuerliga integrationssystem (I2P) eller via Gitlab (I2P+). En Java SDK och Apache Ant krävs för att bygga från källkoden.
  • Engagera dig!
    I2P utvecklas och underhålls främst genom obetalt, frivilligt deltagande av medlemmar i gemenskapen. Mer information om hur du kan delta finns tillgänglig på volontärsidans på den officiella webbplatsen.
  • I2P+
    För mer information om I2P+, besök skank.i2p eller i2pplus.github.io eller anslut till kanalen #saltR på I2P:s IRC-nätverk. Patches för I2P+ är välkomna. Vänligen skicka pull requests via Gitlab.
