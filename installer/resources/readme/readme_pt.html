• Começando
    Se você acabou de iniciar o I2P+, o número de Pares Ativos indicado na seção Pares na barra lateral deve começar a crescer nos próximos minutos, e você também verá um Destino Local chamado Clientes Compartilhados listado na seção Túneis de Serviço, e possivelmente outros clientes e servidores dependendo de como o I2P está configurado. Esses Destinos Locais fornecem conexões para serviços em diferentes portas para a rede I2P, permitindo o funcionamento de BitTorrent, e-mail, proxy web e outros serviços. Seu Banco de Dados de Rede indica todos os pares localmente conhecidos na rede. Você também pode monitorar as conexões de pares existentes, revisar os perfis de desempenho de pares e visualizar os túneis ativos e seu status.
  • Integração de Rede
    Da primeira vez que você iniciar o I2P+, pode levar alguns minutos para que ele seja integrado à rede e encontre pares adicionais para otimizar sua integração, então, por favor, tenha paciência. Quando as estrelas verdes forem indicadas ao lado de seus Destinos Locais na seção Túneis de Serviço na barra lateral, há uma variedade de coisas que você pode fazer com o I2P, e abaixo apresentamos algumas delas.
  • Status de Rede
    Depois que o I2P+ tiver tempo para testar sua conectividade de rede, o indicador de Status de Rede na barra lateral idealmente indicará "OK". Se o status indicar "Firewalled", o I2P+ ainda funcionará, mas o desempenho da rede não será ideal, e sua capacidade de contribuir com largura de banda para a rede será prejudicada. Para um desempenho ideal, considere abrir sua porta externa do I2P+ em seu firewall e roteador fornecido pelo provedor de serviços de internet. Mais informações sobre as mensagens de status estão disponíveis na página de ajuda da Interface do Usuário.
  • Inscrições no Catálogo de Endereços
    Para que o I2P+ seja capaz de resolver automaticamente os nomes de hospedeiros .i2p, ele depende principalmente de inscrições em serviços que fornecem tradução DNS para hosts conhecidos. Se você instalou o I2P+ em vez de fazer um upgrade do I2P, você já terá inscrições suficientes no Catálogo de Endereços configuradas; após a instalação, seu catálogo de endereços deverá ser preenchido com nomes de hospedeiros para serviços ativos na rede sem intervenção. Se você fez um upgrade do I2P, verifique a seção de inscrição no catálogo de endereços nas Perguntas Frequentes para obter ajuda para adicionar inscrições adicionais.
  • Configuração do Navegador
    Para navegar em sites na rede, você precisará configurar o navegador para usar o proxy HTTP padrão configurado na porta 4444, que também fornece um outproxy que permite a navegação anônima em sites da clearnet. Um mini-guia de configuração do Firefox é fornecido na página de Configuração na seção de ajuda, ou consulte a seção abaixo para obter um link para o guia de configuração multi-navegador. Depois de configurar seu navegador e as inscrições no catálogo de endereços, notbob.i2p é um ótimo site para descobrir conteúdo na rede.

Serviços no I2P

  • Navegação na Internet Invisível e Pública
    No I2P, você pode acessar sites hospedados em I2P ("eepsites") ou navegar na web normal de forma anônima através do proxy web incorporado do I2P (outproxy). Configure o seu navegador para usar o proxy HTTP em 127.0.0.1 porta 4444, em seguida, acesse um site de serviço oculto ou um endereço normal não criptografado iniciado por http://. Para desabilitar ou alterar o acesso ao outproxy para a internet normal, remova ou modifique a entrada de outproxy para false.i2p ou purokishi.i2p na seção Outproxies na página de edição do cliente de túnel do Proxy HTTP no Tunnel Manager.
  • E-mail Anônimo
    O sistema de e-mail baseado no I2P chamado Postman pode ser acessado tanto pelo cliente de e-mail incorporado do I2P (Susimail) quanto por qualquer cliente de e-mail padrão. Um gateway de e-mail permite receber e-mails de endereços da internet normal; seu endereço @mail.i2p também pode ser contatado através de @i2pmail.org. Para criar uma conta, visite hq.postman.i2p.
  • Transferência de Arquivos Anônima
    O I2PSnark está integrado ao I2P, fornecendo transferências de BitTorrent anônimas e criptografadas. Alternativamente, há também o BiglyBT (o sucessor do Azureus/Vuze), um cliente BitTorrent que suporta download de torrents sobre o I2P, e o XD, um cliente BitTorrent exclusivo do I2P escrito em go. Outros serviços incluem hospedagem de arquivos baseada em navegador, como o cake.i2p e o fs.i2p.
  • Chat Anônimo
    Inicie o seu cliente de IRC (por exemplo, HexChat) e conecte-se ao servidor em 127.0.0.1 porta 6668. Você não precisa configurar um proxy no seu cliente de IRC; o I2P fornece um túnel IRC local (configurado no seu cliente de IRC como um servidor IRC ou host remoto). Ao entrar, #saltr e #i2p-dev são apenas alguns dos canais disponíveis aos quais você pode desejar se juntar. Também existe um aplicativo de mensagens instantâneas baseado no I2P multiplataforma, disponível no Github.
  • Hospedagem Web Anônima e Criptografada
    O I2P possui um servidor web para hospedar seu próprio site na rede I2P: uma instância do Jetty ouvindo em http://127.0.0.1:7658/. Após iniciar o Túnel do Servidor Web I2P pré-configurado, seu site será visível (mas não localizado) para os outros. Instruções para configurar seu site e registrar um domínio .i2p são fornecidas na seção de ajuda.
  • Plug-ins para o I2P
    Visite a página de plug-ins do zzz para links de download direto de vários plug-ins. Se você é um desenvolvedor, é fornecido um framework agnóstico de linguagem para escrever seus próprios plug-ins.

Informações para desenvolvedores e contribuidores

  • Relatando bugs
    Para relatar um bug, por favor abra uma solicitação em git.idk.i2p. Para discussões relacionadas a desenvolvimento, por favor visite fóruns de desenvolvedores do zzz ou o canal do desenvolvedor (#i2p-dev) na rede IRC do I2P.
  • Desenvolvendo no I2P
    Se você é um desenvolvedor pode considerar escrever suas próprias aplicações I2P ou aprimorar a segurança da sua aplicação aproveitando a rede I2P. Ambos os protocolos UDP e TCP são provisionados e também é possível transmissão em tempo real. Também fornecemos acesso completo à API do Java e as bibliotecas principais do I2P estão hospedadas no repositório Apache Maven. Os desenvolvedores podem navegar o código-fonte no Github (I2P) ou Gitlab (I2P+) e monitorar o status atual de compilação via a plataforma de integração contínua Travis (I2P) ou via Gitlab (I2P+). É necessário um Java SDK e o Apache Ant para compilar a partir do código-fonte.
  • Participe!
    O I2P é desenvolvido e mantido principalmente por meio da participação voluntária da comunidade, sem financiamento. Mais informações sobre como você pode participar estão disponíveis na página de voluntários no site oficial.
  • I2P+
    Para mais informações sobre o I2P+, visite skank.i2p ou i2pplus.github.io, ou junte-se ao canal #saltR na rede IRC do I2P. Patches para o I2P+ são bem-vindos; por favor envie solicitações pull via Gitlab.
