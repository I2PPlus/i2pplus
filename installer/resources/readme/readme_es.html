• Empezando
    Si acabas de iniciar I2P+, el número de Pares Activos indicado en la sección Pares en la barra lateral debería comenzar a crecer en los próximos minutos, y también verás una Destino Local llamado Clientes Compartidos listado en la sección Túneles de Servicio, y posiblemente otros clientes y servidores dependiendo de cómo esté configurado I2P. Estos Destinos Locales brindan conexiones a servicios en diferentes puertos en la red de I2P, permitiendo que funcionen BitTorrent, correo electrónico, el proxy web y otros servicios. Tu Base de Datos de Red indica todos los pares conocidos localmente en la red. También puedes monitorear conexiones de pares existentes, revisar perfiles de rendimiento de pares y ver túneles activos y su estado.
  • Integración en la Red
    La primera vez que inicies I2P+, puede tomar unos minutos para iniciar (integrarte) en la red y encontrar pares adicionales para optimizar tu integración, así que por favor ten paciencia. Una vez que aparezcan estrellas verdes junto a tus Destinos Locales en la sección Túneles de Servicio en la barra lateral, hay varias cosas que puedes hacer con I2P, y a continuación te presentamos algunas de ellas.
  • Estado de la Red
    Una vez que I2P+ haya tenido tiempo para probar tu conectividad de red, el indicador de Estado de la Red en la barra lateral idealmente debería indicar "OK". Si el estado indica "Firewalled", I2P+ seguirá funcionando aunque el rendimiento de la red no será óptimo, y se verá obstaculizada tu capacidad para contribuir ancho de banda a la red. Para un rendimiento óptimo, considera abrir tu puerto externo I2P+ en tu firewall y en el enrutador proporcionado por tu ISP. Se encuentra disponible más información sobre los mensajes de estado en la página de ayuda de la Interfaz de Usuario.
  • Suscripciones a la Libreta de Direcciones
    Para que I2P+ pueda resolver automáticamente los nombres de host .i2p, depende principalmente de suscripciones a servicios que proporcionan traducción DNS para hosts conocidos. Si instalaste I2P+ en lugar de actualizar desde I2P, ya tendrás suficientes suscripciones a la libreta de direcciones configuradas; después de la instalación, tu libreta de direcciones debería llenarse con nombres de host para servicios activos en la red sin intervención. Si actualizaste desde I2P, consulta la sección de suscripciones a la libreta de direcciones en las Preguntas Frecuentes para obtener ayuda añadiendo suscripciones adicionales.
  • Configuración del Navegador
    Para navegar por sitios web en la red, deberás configurar tu navegador para usar el proxy HTTP predeterminado configurado en el puerto 4444, que también proporciona un outproxy que permite la navegación anónima de sitios en clearnet. Se proporciona una mini-guía para configurar Firefox en la página de Configuración de la sección de ayuda, o consulta la sección a continuación para obtener un enlace a la guía de configuración multi-navegador. Una vez que hayas configurado tu navegador y suscripciones a la libreta de direcciones, notbob.i2p es un gran sitio para descubrir contenido en la red.

Servicios en I2P

  • Navegación en la internet invisible y pública
    En I2P puedes acceder a sitios web alojados en I2P ("eepsites") o navegar por la web normal de forma anónima a través del proxy web (outproxy) incorporado en I2P. Configura tu navegador para que use el proxy HTTP en 127.0.0.1 puerto 4444, luego navega a un sitio web de servicio oculto o a una dirección normal no encriptada con http://. Para deshabilitar o cambiar el acceso al outproxy de internet normal, elimina o modifica la entrada del outproxy para false.i2p o purokishi.i2p en la sección Outproxies en la página de edición del túnel del cliente proxy HTTP en el Túnel Manager.
  • Correo Electrónico Anónimo
    El sistema de correo basado en I2P de Postman se puede acceder a través del cliente de correo incorporado de I2P (Susimail) o cualquier cliente de correo estándar. Una pasarela de correo permite recibir correo de direcciones de internet normales; tu dirección @mail.i2p también se puede contactar a través de @i2pmail.org. Para obtener una cuenta, visita hq.postman.i2p.
  • Transferencia de Archivos Anónima
    I2PSnark está integrado en I2P y ofrece transferencias de BitTorrent anónimas y encriptadas. Alternativamente, existe BiglyBT (el sucesor de Azureus/Vuze), un cliente de BitTorrent que admite torrentes a través de I2P, y XD, un cliente de BitTorrent exclusivo de I2P escrito en Go. Los servicios adicionales incluyen hosting de archivos basado en el navegador, como cake.i2p y fs.i2p.
  • Chat Anónimo
    Inicia tu cliente de IRC (por ejemplo, HexChat) y conéctate al servidor en 127.0.0.1 puerto 6668. No necesitas configurar un proxy en tu cliente de IRC; I2P te proporciona un túnel de IRC local (configurado en tu cliente de IRC como un servidor IRC o host remoto). Una vez allí, #saltr y #i2p-dev son solo algunos de los canales disponibles a los que podrías unirte. También existe una aplicación de mensajería instantánea basada en I2P de plataforma cruzada, disponible en Github.
  • Hosting Web Anónimo y Encriptado
    I2P viene con un servidor web para alojar tu propio sitio web en la red de I2P: una instancia de Jetty escuchando en http://127.0.0.1:7658/. Después de iniciar el túnel del Servidor Web I2P preconfigurado, tu sitio será visible (pero no descubrible) para otros. Las instrucciones para configurar tu sitio y registrar un dominio .i2p se proporcionan en la sección de ayuda.
  • Plugins para I2P
    Visita la página de plugins de zzz para descargar varios plugins directamente. Si eres un desarrollador, se proporciona un framework agnóstico de lenguaje para escribir tus propios plugins.

Información para desarrolladores y colaboradores

  • Reportar errores
    Para reportar un error, por favor presenta un ticket en git.idk.i2p. Para discusiones relacionadas con los desarrolladores, por favor visita los foros de desarrolladores de zzz o el canal de desarrolladores (#i2p-dev) en la red IRC de I2P.
  • Desarrollando en I2P
    Si eres un desarrollador, puedes considerar escribir tus propias aplicaciones de I2P o mejorar la seguridad de tu aplicación aprovechando la red I2P. Tanto los protocolos UDP como TCP están provisionados, y también es posible el streaming en tiempo real. También proporcionamos acceso completo a la API de Java, y las bibliotecas principales de I2P están alojadas en el repositorio Apache Maven. Los desarrolladores pueden navegar el código fuente en Github (I2P) o Gitlab (I2P+), y monitorear el estado actual de la compilación a través de la plataforma de integración continua Travis (I2P) o a través de Gitlab (I2P+). Se requiere un Java SDK y Apache Ant para compilar desde el código fuente.
  • ¡Participa!
    I2P se desarrolla y mantiene principalmente a través de la participación voluntaria y sin financiamiento de los miembros de la comunidad. Más información sobre cómo puedes participar está disponible en la página de voluntarios en el sitio web oficial.
  • I2P+
    Para obtener más información sobre I2P+, por favor visita skank.i2p o i2pplus.github.io, o únete al canal #saltR en la red IRC de I2P. Se aceptan parches para I2P+; por favor envía las solicitudes de extracción a través de Gitlab.
