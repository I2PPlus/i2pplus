• Hochfahren
    Wenn Sie gerade I2P+ gestartet haben, sollte die Anzahl der Aktiven Peers unter dem Abschnitt Peers in der Seitenleiste in den nächsten Minuten wachsen. Sie sehen auch ein Lokales Ziel mit dem Namen Shared Clients unter dem Abschnitt Service-Tunnels und möglicherweise auch andere Clients und Server, abhängig von der Konfiguration von I2P. Diese Lokalen Ziele ermöglichen Verbindungen zu Diensten auf verschiedenen Ports des I2P-Netzwerks, wodurch BitTorrent, E-Mail, der Web-Proxy und andere Dienste funktionieren können. Ihre Netzwerkdatenbank zeigt alle lokal bekannten Peers im Netzwerk an. Sie können auch bestehende Peerverbindungen überwachen, Leistungsprofile der Peers überprüfen und aktive Tunnel und deren Status anzeigen.
  • Netzwerkintegration
    Beim ersten Start von I2P+ kann es einige Minuten dauern, bis Sie in das Netzwerk integriert werden und zusätzliche Peers finden, um Ihre Integration zu optimieren. Bitte haben Sie Geduld. Sobald grüne Sterne neben Ihren Lokalen Zielen unter Service-Tunnels in der Seitenleiste angezeigt werden, haben Sie verschiedene Möglichkeiten, die Sie mit I2P tun können. Im Folgenden stellen wir Ihnen einige davon vor.
  • Netzwerkstatus
    Wenn I2P+ ausreichend Zeit hatte, Ihre Netzwerkverbindung zu testen, sollte der Netzwerkstatus in der Seitenleiste idealerweise "OK" anzeigen. Wenn der Status "Firewalled" anzeigt, funktioniert I2P+ immer noch, jedoch ist die Netzwerkperformance nicht optimal und Ihre Fähigkeit, Bandbreite zum Netzwerk beizutragen, ist eingeschränkt. Für optimale Leistung sollten Sie in Erwägung ziehen, Ihren externen I2P+ Port in Ihrer Firewall und Ihrem vom Internetdienstanbieter bereitgestellten Router zu öffnen. Weitere Informationen zu den Statusmeldungen finden Sie auf der Hilfeseite zur Benutzeroberfläche.
  • Adressbuch-Abonnements
    Damit I2P+ automatisch .i2p Hostnamen auflösen kann, ist es hauptsächlich auf Abonnements von Diensten angewiesen, die DNS-Übersetzungen für bekannte Hosts bereitstellen. Wenn Sie I2P+ installiert haben und nicht von I2P aktualisiert haben, haben Sie bereits ausreichende Adressbuch-Abonnements konfiguriert. Nach der Installation sollten Ihre Adressbuch automatisch mit Hostnamen für aktive Dienste im Netzwerk gefüllt werden. Wenn Sie von I2P aktualisiert haben, überprüfen Sie den Abschnitt Adressbuch-Abonnements in den FAQ, um Hilfe beim Hinzufügen zusätzlicher Abonnements zu erhalten.
  • Browserkonfiguration
    Um Websites im Netzwerk zu durchsuchen, müssen Sie Ihren Browser so konfigurieren, dass er den standardmäßigen HTTP-Proxy auf Port 4444 verwendet. Dieser Proxy bietet auch einen Outproxy, der anonymes Durchsuchen von Clearnet-Sites ermöglicht. Eine Minianleitung zur Konfiguration von Firefox finden Sie auf der Konfigurationsseite im Hilfebereich oder verwenden Sie den Link unten für eine Anleitung zur Einrichtung eines Mehrbrowser-Setups. Nachdem Sie Ihren Browser und Ihre Adressbuch-Abonnements konfiguriert haben, ist notbob.i2p eine großartige Seite zum Entdecken von Inhalten im Netzwerk.

Dienstleistungen auf I2P

  • Unsichtbares Internet & öffentliches Web-Browsing
    Auf I2P können Sie auf I2P-gehostete Websites ("Eepsites") zugreifen oder das normale Web anonym über den integrierten Webproxy (Outproxy) von I2P durchsuchen. Konfigurieren Sie Ihren Browser, um den HTTP-Proxy bei 127.0.0.1 Port 4444 zu verwenden und dann eine versteckte Service-Website oder eine normale, unverschlüsselte http://-Adresse aufzurufen. Um den Zugriff auf das normale Internet über den Outproxy zu deaktivieren oder zu ändern, entfernen oder modifizieren Sie den Eintrag für false.i2p oder purokishi.i2p im Abschnitt Outproxies auf der Bearbeitungsseite des HTTP-Proxy-Client-Tunnels im Tunnel Manager.
  • Anonyme E-Mail
    Das I2P-basierte E-Mail-System von Postman kann entweder über den integrierten Mail-Client von I2P (Susimail) oder über jeden Standard-Mail-Client erreicht werden. Ein Mail-Gateway ermöglicht den Empfang von E-Mails von normalen Internetadressen; Ihre @mail.i2p-Adresse ist auch über @i2pmail.org kontaktierbar. Um ein Konto zu erstellen, besuchen Sie hq.postman.i2p.
  • Anonymer Dateitransfer
    I2PSnark ist in I2P integriert und ermöglicht anonyme, verschlüsselte BitTorrent-Übertragungen. Alternativ gibt es BiglyBT (der Nachfolger von Azureus/Vuze), ein BitTorrent-Client, der Torrents über I2P unterstützt, und XD, ein nur auf I2P basierender BitTorrent-Client, der in Go geschrieben ist. Weitere Dienste umfassen browserbasiertes Datei-Hosting wie cake.i2p und fs.i2p.
  • Anonymer Chat
    Starten Sie Ihren IRC-Client (z. B. HexChat) und verbinden Sie sich mit dem Server bei 127.0.0.1 Port 6668. Sie müssen keinen Proxy in Ihrem IRC-Client konfigurieren; I2P stellt Ihnen einen lokalen IRC-Tunnel zur Verfügung (konfiguriert in Ihrem IRC-Client als IRC-Server oder Remote-Host). Sobald Sie dort sind, können Sie Kanälen wie #saltr und #i2p-dev beitreten. Es gibt auch eine plattformübergreifende I2P-basierte Sofortnachrichten-Anwendung, die auf Github verfügbar ist.
  • Anonymes verschlüsseltes Webhosting
    I2P enthält einen Webserver, um Ihre eigene Website im I2P-Netzwerk zu hosten: eine Jetty-Instanz, die unter http://127.0.0.1:7658/ lauscht. Nach dem Starten des vorab konfigurierten I2P-Webserver-Tunnels ist Ihre Website für andere sichtbar (aber nicht auffindbar). Anweisungen zur Einrichtung Ihrer Website und zur Registrierung einer .i2p-Domain finden Sie im Hilfebereich.
  • Plugins für I2P
    Besuchen Sie zzz's Plugins-Seite für direkte Download-Links zu verschiedenen Plugins. Wenn Sie ein Entwickler sind, steht Ihnen ein sprachunabhängiges Framework zur Verfügung, um Ihre eigenen Plugins zu schreiben.

Informationen für Entwickler & Mitwirkende

  • Bugs melden
    Um einen Fehler zu melden, bitte ein Ticket auf github.com/I2PPlus/i2pplus/issues erstellen. Für entwicklerbezogene Diskussionen besuchen Sie bitte zzz's Entwicklerforen oder den Entwicklerkanal (#i2p-dev) im I2P-IRC-Netzwerk.
  • Entwicklung auf I2P
    Wenn Sie ein Entwickler sind, könnten Sie Ihre eigenen I2P-Anwendungen schreiben oder die Sicherheit Ihrer Anwendung durch Nutzung des I2P-Netzwerks verbessern. Sowohl das UDP- als auch das TCP-Protokoll sind bereitgestellt, und auch Echtzeit-Streaming ist möglich. Wir stellen auch vollen Zugriff auf die Java API zur Verfügung, und die Kern-I2P-Bibliotheken werden im Apache Maven Repository gehostet. Entwickler können den Quellcode auf Github (I2P) oder Gitlab (I2P+) durchstöbern und den aktuellen Build-Status über die Travis-kontinuierliche Integration (I2P) oder über Gitlab (I2P+) überwachen. Zum Kompilieren ist ein Java SDK und Apache Ant erforderlich.
  • Machen Sie mit!
    I2P wird hauptsächlich durch die unbezahlte, freiwillige Teilnahme von Gemeindemitgliedern entwickelt und gepflegt. Weitere Informationen zur Teilnahme finden Sie auf der Mitmachseite auf der offiziellen Website.
  • I2P+
    Für weitere Informationen über I2P+ besuchen Sie bitte skank.i2p oder i2pplus.github.io oder treten Sie dem Kanal #saltR im I2P IRC-Netzwerk bei. Patches für I2P+ sind willkommen. Bitte reichen Sie Pull-Anfragen über Gitlab ein.
