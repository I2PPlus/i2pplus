Core tunnel management, message processing, and encryption for I2P's anonymous routing system.

This package provides the complete implementation of I2P's tunnel system, which forms the backbone of anonymous communication in the I2P network. Tunnels provide bidirectional, encrypted paths through multiple routers, ensuring both anonymity and message delivery reliability.

Key Components:

  • TunnelDispatcher - Central coordinator for tunnel message routing
  • TunnelGateway - Manages tunnel entry and exit points
  • TunnelParticipant - Handles participation in tunnels
  • TunnelCreator - Manages tunnel creation and setup
  • TunnelPool - Manages collections of tunnels
  • TunnelInfo - Contains tunnel metadata and configuration
  • HopProcessor - Processes messages at each tunnel hop
  • TunnelValidator - Validates tunnel integrity and authenticity

Main Functionality:

  • Tunnel creation and management
  • Message encryption and decryption at each hop
  • Five tunnel hop types: IBGW, IBEP, OBEP, OBGW, Middle
  • Message validation and authentication
  • Queueing and throttling for performance management
  • Tunnel failure detection and recovery
  • Message routing and forwarding
  • Tunnel statistics and monitoring

Tunnel Hop Types:

  • Inbound Gateway (IBGW) - Entry point for inbound tunnels
  • Inbound Endpoint (IBEP) - Exit point for inbound tunnels
  • Outbound Endpoint (OBEP) - Entry point for outbound tunnels
  • Outbound Gateway (OBGW) - Exit point for outbound tunnels
  • Middle Hop - Intermediate router in tunnel path

Usage Notes:

  • Tunnel building operations are handled in the pool package
  • Essential for I2P's anonymity and routing capabilities
  • Integrates with transport layer for message delivery
  • Provides the foundation for client communication
