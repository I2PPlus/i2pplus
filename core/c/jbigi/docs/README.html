JBIGI Build Guide

JBIGI is the native BigInteger library used by I2P for fast cryptographic operations. This guide covers how to build it from source.

Quick Start

cd core/c/jbigi
./build.sh help

Prerequisites

  • Java JDK with JNI headers (for jni.h)
  • GNU Compiler Collection (GCC)
  • m4 macro processor
  • wget (to download GMP)
Note: Java 17+ is required for recent I2P versions.

Build Commands

Build for current machine

cd core/c/jbigi
./build.sh local

Builds a libjbigi.so optimized for your current CPU architecture.

Build all architectures

cd core/c/jbigi
./build.sh all

Builds optimized libraries for all supported CPU types (zen, skylake, core2, arm, etc.).

Preview what would be built

cd core/c/jbigi
./build.sh all --dry-run

Shows the list of architectures without actually building.

Dynamic build (requires system GMP)

cd core/c/jbigi
./build.sh local dyn

Links against system GMP library instead of static linking. Faster build but requires GMP installed.

Environment Variables

  • BITS=32 or BITS=64 - Force 32 or 64 bit build
  • TARGET=linux|windows|freebsd|darwin - Target OS
  • CC=gcc - C compiler
  • JAVA_HOME=/path/to/jdk - Path to JDK

Examples

Build for current machine

./build.sh local

Build 32-bit

BITS=32 ./build.sh local

Cross-compile for Windows 64-bit

TARGET=mingw64 CC=x86_64-w64-mingw32-gcc BITS=64 ./build.sh all

Cross-compile for FreeBSD

TARGET=freebsd BITS=64 ./build.sh all

Output Location

Built libraries are copied to:

core/c/jbigi/lib/

From there, copy to your I2P installation's lib/ directory.

Troubleshooting

"Cannot find jni.h"

Set JAVA_HOME to your JDK installation:

export JAVA_HOME=/usr/lib/jvm/java-17-openjdk-amd64

"m4 not found"

Install m4:

sudo apt-get install m4    # Debian/Ubuntu
sudo yum install m4        # RHEL/CentOS

"Unsupported platform"

Your CPU architecture may not be supported by GMP. Check the supported platforms in build-all.sh.

Supported Architectures

  • x86_64: zen3, zen2, zen, skylake, core2, atom, bulldozer, and more
  • x86 (32-bit): pentium, athlon, and more
  • ARM: armv5, armv6, armv7a, armv8 (aarch64)
Note: Zen4 and Zen5 are not supported by GMP 6.3.0. These CPUs will fall back to the zen3 binary.
