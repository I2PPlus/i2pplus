EDNS(0) extension mechanisms for DNS.

This package implements the Extension Mechanisms for DNS (EDNS(0)) as defined in RFC 6891. EDNS(0) provides a framework for extending DNS capabilities beyond the original limitations, including larger message sizes, additional record types, and extended flags.

Key features:

  • EDNS option parsing and serialization
  • Extended RCODE and flag handling
  • UDP payload size negotiation
  • Cookie options for security
  • NSID and other experimental options

EDNS(0) is essential for modern DNS operations, particularly for DNSSEC validation and responses that exceed the traditional 512-byte limit.
